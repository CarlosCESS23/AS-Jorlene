# Apresentação de Modelo Cascata

## O que seria Ciclos de vida de Desenvolvimento de Software (CVDS)

<li> Processo empregado no projeto, desenvolvimento e Teste de software, o seu objetivo é produzir software mais alta qualidade, menor tempo possivel e tendo seu custo baixo

<li>Existe vários modelos de CVDS, como por exemplo dos mais comuns: </li>
<ol>
  <li>Modelo Cascata</li>
  <li>Modelo Interativo</li>
  <li>Modelo Espeiral</li>
  <li>Modelo V</li>
  <li>Modelo Big Bang</li>
</ol>

## Modelo em cascata (WaterFall)

Ou conhecido como modelo de ciclo de vida linear/sequencial/clássico

<li>O modelo que no qual as atividades são realizadas em forma sistematica (Em sequencial), tendo a cada fase, ou seja, cada fase tem seu caso especifico e deve-se completar para ir na próxima fase</li>
<li>Foi uma das primeiras abordagens usadas no desenvolvimento de software</li>
<li>O processo é tratado como fluxo sequencial,linear de operações</li>

### Fases do modelo em Cascata

#### Requisito/Levantamento da Análise do Requisito

É a 1° fase, levanta todos os requisitos do sistema ( TODOS MESMO ) , que são capturados e criar o documento o que será feito durante nas próximas fases

#### Projeto

Fase seguinte é o projeto, as especificações e os requisitos vão ser estudado, de como elaborar e desenvolver durante o tempo, mas não somente de software mas também o hardware , que ajuda a definir a arquitetura do sistema (Como se fosse uma planta do sistema)

#### Implementação

Fase da codificação, ou é a fase do desenvolvimento desse software

#### Fase de Teste de Software

Tudo que foi desenvolvido nessa fase anterior, vai ser tudo testado para analisar se está tudo de acordo com os requisito e realmente fazendo o que deveria acontecer, se houver erro, vai ser corrigido nessa parte

#### Integração/implementação

Depois da fase do Teste, seria que o sistema é implantado para que o usuário possa usar

#### Manuntenção

O final dessa fase, que seria atualização do sistema, para manter o sistema funcional para o sistema que foi implementado ao cliente

### Quando o modelo é aplicado?

O modelo em cascata se aplica quando o software a ser desenvolvido se encaixar em um desses cenários especificos

<li>Quando o produto é estavel, sem previsão de mudançãs ao longo do processo</li>

<li>Quando a tecnologia é bem compreendida e não é dinâmica</li>

<li>Projetos mais longos podem apresentar problemas, por isso que fica bom para projetos que são rápidos em desenvolver</li>

### Quais são algumas vantagens?

<li> Para modelo simples, fica fácil de entender e usar, tem vantagens para projetos pequenos, pois os resquisitos são muito bens compreendido</li>

<li>Software utilizavel ( Ou seja, o cliente somente pode utilizar só no final da fase)</li>
<li>Possui seu Riscos e incertezas, por isso não é aconselhado utilizar para projeto longo ou mt grandes</li>
<li>Durante as etapas, é complicado medir o desenvolvimento</li>
<li>Mudanças drasticas durante o ciclo de vida, podem inclusive levar ao fim do projeto</li>
<li>A integração só é feita só uma vez, no final do processo, o que não permite detectar determinados tipos de problemas com antecedência</li>

## Exemplo prático para demostração modelo cascata

<strong> Desenvolvimento de um Sistema de Controle de Estoque para um Supermercado</strong>

  <li>Requisito:</li>
  <li>Projeto:</li>
  <li>Implementação:</li>
  <li>Teste:</li>
  <li>Implantação:</li>
  <li>Manutenção:</li>
</ol>

### 1. Requisito

Supermercado fornece uma lista detalhada o que ela deseja:

<li> Cadastro de produto</li>
<li>Controle de entrada e saída de estoque</li>
<li>Geração de relatório sobre a respeito de estoque dos produtos</li>
<li>Integração com sistema de venda</li>

A fase do requisito são documentados e aprovados antes de seguir para a próxima fase

### 2. Projeto

Nessa fase, é os desenvolvedores vão analisar o requisito e realmente começar a criar a arquitetura do sistema, com banco de dados, interfaces e fluxos de dados e layouts de telas é definido
<br> Após isso, caso seja aceito a aprovação, ele simplesmente avança para próxima fase

### 3. Implementação

Nessa fase, os desenvolvedores começam a desenvolver o que haviam decidido a utilização da fase anterior, onde eles escrevem o código do sistema

### 4.Teste

Após a conclusão da implementação, a equipe de teste, testam o programa se realmente está seguindo de acordo com a 1° a primeira fase
